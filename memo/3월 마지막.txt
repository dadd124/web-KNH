
사용자가 직접 정의하면 컴파일러가 만든 기본생성자는 사용되지 않는다.

메서드처럼 매개변수를 가질 수 있다.

메서드(계산기)
정의(기능을 만드는 것)
-> 숫자 두 개와 연산자를 받을 준비
호출
-> 숫자와 연산자를 전달하여 연산결과를 돌려받는다.

return
매개변수, 메서드 내부에서 정의된 변수는 지역변수.
메서드가 호출되어 실행한 다음 종료되면 메모리에서 삭제된다.

만약, 메서드 밖에서 연산결과를 사용하고 싶다면 return을 통해 값을 밖으로 꺼내야 한다.

오버로딩(overloading)
- 같은 이름으로 서로 다른 개수, 타입의 매개변수를 각각 처리하는 방법

- 반환형은 달라도 상관없음
- 메서드(or 생성자)명은 같아야 한다.
- 매개변수의 개수가 다르거나, 타입이 다르거나, 순서가 달라야 한다.

-----------------

접근제한자(Access Modifier)
public : 프로젝트 내에서는 어디서든 사용 가능
default : 같은 패키지 내의 다른 클래스 어디서든지 사용 가능
private : 현재 클래스에서만 사용 가능
protected

필드/생성자/메서드/클래스에 붙힐 수 있다.

상속의 특징
하나의 부모는 여러 클래스한테 상속할 수 있다.
하나의 자식이 여러 클래스를 상속받는 것은 불가능하다.

final
값을 넣은 다음 변경할 수 없다.

static final int FEE = 1000; (상수)

패키지
단순히 클래스를 분리하는 용도가 아니라 클래스별 구분자 역할도 한다.
패키지도 계층구조로 만들 수 있다.
com
-> kumho
-> hankook

도트(.)를 기준으로 계층이 생긴다.

싱클톤 패턴
객체를 메모리에 한 번만 올리고, 올라간 객체를 계속 사용하는 방식
1. 생성자를 private으로 정의해서 외부에서 호출하지 못하도록 한다.
2. 클래스 내부에서 static으로 객체를 하나 생성한다.(private)
3. 클래스 내부에서 만든 객체를 사용할 수 있도록 public으로 된 메서드를 만든다.

상속
부모 클래스의 필드나 메서드를 자식 클래스에서 사용할 수 있는 것
자식 클래스가 상속받을 클래스를 고른다.

클래스명 extends 상속 받을 클래스명{
